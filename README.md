# Fault-Aware Decision Support for Renewable-Powered EV Charging Stations

This repository contains the reproducible synthetic benchmark pipeline used for the manuscript:

**Fault-Aware Decision Support for Renewable-Powered EV Charging Stations Using Multi-Source Explainable Learning**

## Scope

The dataset generated by this repository is simulation-based. It is not a measured field EVCS dataset. The code is intended to reproduce the scenario-controlled benchmark, fault-injection experiments, feature extraction, diagnostic modelling, explainability analysis, maintenance-oriented risk mapping, and scenario-based operational-response results reported in the paper.

## Repository contents

```text
src/run_evcs_fault_aware_pipeline.py   # main reproducible pipeline
config/random_seeds.json               # fixed random seeds used in the benchmark
config/nominal_config.json             # short summary of benchmark settings
outputs/                               # generated outputs after running the pipeline
requirements.txt                       # Python package requirements
CITATION.cff                           # citation metadata for GitHub and Zenodo
.zenodo.json                           # Zenodo metadata
```

## How to reproduce

Create a Python environment and install the required packages:

```bash
pip install -r requirements.txt
```

Run the full benchmark pipeline:

```bash
python src/run_evcs_fault_aware_pipeline.py
```

The script writes generated data, tables, figures, and trained model files to:

```text
outputs/data/
outputs/tables/
outputs/figures/
outputs/models/
```

## Reproducibility

The nominal benchmark uses a fixed random seed of 42. Repeated-seed stability analysis uses the seed set [42, 43, 44, 45, 46]. The main numerical assumptions are collected in the `Config` class inside `src/run_evcs_fault_aware_pipeline.py`.
